# 输入校验（Input Validation）

SKILL.md 中「输入校验」章节的完整规则，供 Agent 按需加载。

## 概述

输入校验分为两个层次：

1. **边界条件守卫（强制）**：检测输入是否为英文、是否包含完整句子。命中即中止分析，**必须执行**。
2. **新闻文本判定（可选）**：检测输入是否为新闻文本。可运行预校验脚本辅助判断，不影响核心分析流程。

## 边界条件守卫（强制，优先执行）

在进入事实核查和分析流程**之前**，必须先检查以下两项边界条件。任一条件命中即**中止分析**并向用户输出对应提示。详细规则见 [edge-cases.md](edge-cases.md)「🛑 边界条件守卫」一节。

### 守卫 1：非英文输入检测

**内联判断方法**（脚本不可用时）：
- 目测输入文本的主体字符是否为非拉丁字母（中文、日文、韩文、阿拉伯文、西里尔字母等）
- 如英文单词占比低于总 token 数的 30%，判定为非英文输入

**处理**：拒绝执行分析，提示用户本 skill 专为英文新闻精读设计。

### 守卫 2：英文单词/短语而非完整句子检测

**内联判断方法**（脚本不可用时）：
- 检查输入是否包含句末标点（`.` `!` `?`）
- 如全文无句末标点，且不符合新闻标题特征（3-15 词、首字母大写、含动词），判定为单词/短语输入

**处理**：拒绝执行分析，提示用户提供至少一句完整的英文句子。

## 调用方式（新闻文本判定，可选）

### 方式一（推荐）：stdin 管道输入

```bash
echo "<用户输入内容>" | python scripts/validate-input.py
```

### 方式二：文件路径参数

```bash
python scripts/validate-input.py <文件路径>
```

## 退出码与后续操作

| 退出码 | 含义 | Agent 处理方式 |
|--------|------|---------------|
| `0` | 疑似新闻文本 | 正常执行分析 |
| `1` | 不确定 | 在输出开头添加「❓ 内容属性待确认」提示 |
| `2` | 非新闻文本或边界条件拦截 | 边界条件拦截时直接拒绝并提示用户；非新闻文本时回复「该内容疑似非新闻文本，是否仍要执行语言分析？[是/否]」 |

## 内联判断规则（脚本不可用时的降级方案）

如无法执行脚本，按以下优先级依次判断：

**第一优先级——边界条件**：
1. **语言检测**：输入主体是否为非英文字符？→ 拒绝，提示需要英文输入
2. **句子完整性**：输入是否为零散英文单词/短语（无句末标点、无主谓结构）？→ 拒绝，提示需要完整句子

**第二优先级——新闻特征**：
3. **消息源标识**：检测是否包含已知新闻源（Reuters、BBC、AP 等），完整列表见 [news-sources.md](news-sources.md)
4. **新闻高频动词**：`reports`、`announced`、`stated`、`confirmed`、`according to`
5. **时效性词汇**：`today`、`yesterday`、`this week`、具体星期几
6. **虚构标记词**：`fiction`、`scenario`、`hypothetical`、`AI-generated` — 检测到则触发事实核查

## 脚本说明

`scripts/validate-input.py` 是一个独立的 CLI 工具，基于正则匹配和置信度评分判断输入是否为新闻文本。它会：

- 优先执行边界条件守卫（非英文检测、句子完整性检测），命中即返回退出码 `2`
- 动态加载 `references/news-sources.md` 中的消息源列表
- 检测正向特征（消息源、新闻动词、时效性词汇、标题格式、地名+事件组合）
- 检测负向特征（未来日期、虚构标记词、非新闻文体）
- 返回置信度评分和建议操作
