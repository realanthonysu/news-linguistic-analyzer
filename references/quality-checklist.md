# ✅ 新闻英语分析质量检查清单

> 模型生成输出前，请逐项自检。用户也可用此清单评估输出质量。

## 🔹 基础完整性（必检）
- [必检] 边界条件守卫已在分析前执行（非英文输入检测 + 句子完整性检测）
- [ ] 中文翻译覆盖原文全部信息点，无遗漏关键数据/人名/地名
- [ ] 翻译信息点校验已执行，校验结果已附在译文末尾
- [必检] 翻译信息点校验结果已输出，且发现的遗漏已在译文中补全
- [ ] 用词分析 ≥5 项，且包含至少 1 个隐喻/文化负载词
- [ ] 句子拆解选取了含从句/分词的典型长句（非简单句）
- [ ] 语法分析涵盖时态/语态/消息源位置中至少 2 项
- [ ] 翻译技巧 ≥3 项，且每项含「示例+原理」双要素
- [ ] 延伸思考标注 🌍，且内容具批判性/反思性

> 注意：简洁模式（brief）下用词分析限 3 项、翻译技巧限 2 项，不适用上述最低数量要求

## 🔹 专业准确性（重点）
- [必检] 领域自适应注释已正确注入（触发词匹配、优先级规则、负面场景排除）
- [ ] 专有名词翻译准确（如 `Strait of Hormuz` → 霍尔木兹海峡）
- [ ] 时态逻辑还原正确（现在完成时→"已.../持续..."）
- [ ] 被动语态处理符合中文习惯（避免"被"字堆砌）
- [ ] 消息源位置按中文习惯前置（"据...报道"）
- [ ] 数据/伤亡表述克制（"遇难"替代"死亡"，"致...人"替代"杀死"）

## 🔹 事实核查（关键）
- [ ] 检测日期：如晚于系统当前时间，已添加⚠️提示
- [ ] 检测人物/事件：如与已知事实冲突，已添加⚠️提示
- [ ] 未将预测性内容表述为既定事实
- [ ] 如搜索结果存在单一信源/多源矛盾/细节不一致，已使用「⚠️ 事实核查（低置信度）」格式而非标准格式

## 🔹 用户体验（优化）
- [ ] 表格列宽适中，无换行断裂
- [ ] 技术术语首次出现保留英文+括号解释
- [ ] 段落间有清晰分隔，视觉层次分明
- [ ] 如用户指令含"简洁"，已按规则精简输出
- [ ] 如用户同时发出多个模式指令，已按优先级执行（仅需翻译 > 简洁模式 > 跳过延伸 > 默认完整模式）

## 🔹 伦理与边界（底线）
- [ ] 未对冲突/灾难事件使用情绪化渲染词汇
- [ ] 未添加原文无依据的主观评论
- [ ] 延伸思考聚焦"如何理解/翻译"，而非"事件对错"

## 🔄 批量处理附加检查

完整质量检查项见 [batch-processing.md](batch-processing.md)「质量检查」章节。

**快速对照**：
- [ ] 多段新闻已正确分割并编号
- [ ] 每段分析结构完整，无交叉混淆
- [ ] 末尾含「📊 批量分析小结」

## 📏 超长文本附加检查（>2000 词）

完整质量检查项见 [long-text-processing.md](long-text-processing.md)「质量检查清单」章节。

**快速对照**：
- [ ] 分段切分点位于自然段落边界或主题转换处
- [ ] 每段长度在 400-800 词范围内（允许 ±10% 弹性）
- [ ] 第 2 段起均包含「📎 上文衔接」行
- [ ] 各段分析结构完整，无跨段信息混淆
- [ ] 末尾含「📊 全文综述」
- [ ] 同一术语在不同段落的译法保持一致

> 💡 自检通过后，方可输出最终结果。如有任一 [必检] 项未通过，请优先修正。
