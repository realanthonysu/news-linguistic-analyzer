# 边缘情况处理（Edge Cases & Fallback Strategies）

SKILL.md 中「边缘情况处理」和「错误处理与降级策略」章节的完整规则，供 Agent 按需加载。

## 🛑 边界条件守卫（前置拦截，优先于一切分析步骤执行）

在收到用户输入后、进入事实核查和 6 步分析流程**之前**，必须先执行以下两项边界条件检测。任一条件命中即**中止分析流程**，向用户输出对应提示并结束。

### 守卫 1：非英文输入检测

**判定规则**：输入文本的主体字符为非拉丁字母（如中文汉字、日文假名、韩文字母、阿拉伯文、西里尔字母等），或英文单词占比低于总字符数的 30%（拉丁字母占比）。

**处理策略**：**拒绝执行分析**，输出以下提示：

```text
🚫 输入语言不匹配：本 skill 专为英文新闻精读设计（用词分析、句法拆解、翻译技巧提炼等核心步骤均依赖英文原文）。当前输入为非英文文本，无法执行分析流程。

👉 请提供一段英文新闻文本（如 Reuters、BBC、AP、CNN、Bloomberg 的报道段落）后再使用本 skill。
```

### 守卫 2：英文单词/短语而非完整句子检测

**判定规则**：输入虽为英文，但缺乏基本句子结构。满足以下条件之一即命中：
- 全文无句号（`.`）、问号（`?`）、感叹号（`!`）等句末标点
- 全文不含主谓结构（即找不到 `主语 + 动词` 的基本搭配）
- 英文单词数量 ≥1 但 <5，且无句末标点（排除合理的新闻标题——标题虽短但通常含大写首字母+动词，见下方例外）

**处理策略**：**拒绝执行分析**，输出以下提示：

```text
🚫 输入内容过少或不完整：您的输入看起来是零散的英文单词/短语，而非完整的句子或段落。本 skill 需要至少一句完整的英文句子才能进行有效的语言学分析（句法拆解、翻译技巧提炼等步骤需要完整的句子结构）。

👉 请提供至少一句完整的英文句子，最好是一段英文新闻正文（推荐 50-500 词）。
```

**例外——纯标题输入**：如输入虽短（3-15 词）但符合新闻标题特征（首字母大写、含动词、无句末标点），按「纯标题无正文」处理（输出简短翻译 + 说明正文缺失），不在此守卫拦截范围内。

### 守卫执行顺序

```
用户输入 → 守卫 1（非英文？）→ 守卫 2（仅单词/短语？）→ 事实核查 → 6 步分析
                ↓ 命中                 ↓ 命中
           拒绝 + 提示             拒绝 + 提示
```

## 边缘情况处理

通过边界守卫后，在分析过程中遇到以下边缘情况按对应策略处理：

| 情况 | 处理策略 |
|------|---------|
| 输入为空或极短（<10 词） | 提示用户提供完整新闻段落（至少一句完整的英文句子，最好是一段新闻正文） |
| 英文单词/短语而非完整句子 | 识别到输入仅含零散英文单词或短语（无主谓结构、无标点句末），**拒绝执行分析**，提示用户：「您的输入看起来是零散的英文单词/短语，而非完整的句子或段落。本 skill 需要至少一句完整的英文句子才能进行有效的语言学分析，最好提供一段英文新闻正文。」 |
| 多语言混合文本 | 仅分析英文部分，中文内容原样保留 |
| 纯标题无正文 | 输出简短翻译 + 说明「正文缺失，无法进行深度分析」 |
| 非新闻文体（诗歌、小说、日记） | 识别后提示文体类型，询问是否仍要执行语言分析 |
| 含大量专业术语 | 补充术语表（Glossary），先解释后分析 |
| 消息源无法识别 | 不强制标注来源，在语法分析中说明「原文未标注消息源」 |
| 纯非英语输入（中文、法语、日语等） | **拒绝执行分析**，明确提示用户：「本 skill 专为**英文新闻精读**设计，当前输入为非英文文本，无法执行英文语言学分析流程（用词分析、句法拆解、翻译技巧提炼等均依赖英文原文）。请提供一段英文新闻文本后再使用本 skill。」 |
| 中英文混合输入（英文为主但含中文说明） | 提取英文部分进行分析，忽略中文说明文字；如英文部分不构成完整句子，按「英文单词/短语而非完整句子」处理 |
| 含 HTML/Markdown 标记 | 分析前先在内部剥离标记（`<p>`、`<strong>`、`[link](url)` 等），提取纯文本后执行正常分析流程；不在输出中保留原始标记，翻译时还原为自然文本 |

## 错误处理与降级策略

| 问题 | 降级策略 |
|------|---------|
| 术语歧义（一词多义） | 列出所有可能译法，说明语境选择依据 |
| 文化负载词无直接对应 | 采用「音译+注释」或「意译+原文保留」策略 |
| 长句结构过于复杂 | 拆分为 2-3 个子句分别分析，避免信息过载 |
| 无法确定领域背景 | 跳过领域自适应步骤，在延伸思考中说明 |
| 翻译存在多种合理版本 | 提供主推译法 + 1 个替代方案，说明适用场景差异 |

> 联网验证失败时的降级策略（按优先级依次尝试）：
> 1. **web search**（首选）：执行双语关键词查询
> 2. **web fetch**（次选）：当 web search 不可用时，访问已知权威新闻源页面
> 3. **离线标注**（兜底）：当 web search 和 web fetch 均不可用时，标注「⚠️ 未能联网验证，以下分析基于模型内置知识」
