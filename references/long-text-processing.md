# 超长文本处理（Long-Text Processing）

SKILL.md 中「超长文本处理」章节的完整规则，供 Agent 按需加载。

## 触发条件

当单篇新闻超过 **2000 词**时，采用**主题感知分段策略**，避免机械切割导致语义断裂。

## 分段原则

1. **优先在自然段落边界切分**：空行、小标题、段落缩进处为首选切分点
2. **次选在主题转换处切分**：以下过渡词/短语标志新主题的起点，应作为下一段首句保留：
   - 转折：`However`, `Nevertheless`, `On the other hand`, `By contrast`
   - 递进/并列：`Meanwhile`, `In addition`, `Furthermore`, `Separately`
   - 时间推移：`Later`, `On Monday/Tuesday...`, `The next day`
   - 焦点切换：`In Washington`, `In Tehran`, `Back in London`（地点切换）
3. **单段目标长度 400-800 词**：低于 400 词时与相邻段落合并，超过 800 词时在句号处二次切分
4. **禁止在句子中间切分**：切分点必须位于完整句子末尾

## 上下文衔接

每段分析（第 2 段起）的翻译输出前，添加一行衔接说明：

```text
📎 上文衔接：[前段核心结论，1-2 句概括]
```

衔接说明的作用：
- 为当前段的用词分析和语法分析提供语境锚点
- 避免因分段导致代词指代、时态延续等跨段语言现象丢失
- 帮助用户理解段落间的逻辑关系

## 输出结构

```markdown
## 【第 1 段】[主题概括标题]

### 📝 中文翻译
[分段译文]
✅ 翻译校验通过：原文 N 个关键信息点均已体现

### 🔍 用词分析
...

（完整 6 步分析）

---

## 【第 2 段】[主题概括标题]

📎 上文衔接：前段报道了 X 事件的核心进展与各方反应

### 📝 中文翻译
...

（完整 6 步分析）

---

## 📊 全文综述

### 主题脉络
[各段主题之间的逻辑关系：并列/递进/转折/因果]

### 共性翻译难点
[跨段反复出现的语言现象，如：同一术语在不同语境的译法差异]

### 语言特征总结
[全文层面的时态分布、消息源使用模式、修辞策略等]
```

## 与批量处理的区别

| 维度 | 批量处理 | 超长文本处理 |
|------|---------|------------|
| 输入性质 | 多篇独立新闻 | 一篇长文 |
| 分段依据 | `---` 或空行（用户已分隔） | 主题感知自动切分 |
| 段间关系 | 各篇独立，无衔接 | 上下文衔接 + 全文综述 |
| 术语处理 | 各篇独立分析 | 全文综述中汇总跨段术语一致性 |

## 质量检查清单（超长文本附加项）

- [ ] 分段切分点位于自然段落边界或主题转换处，未在句子中间截断
- [ ] 每段长度在 400-800 词范围内（允许 ±10% 弹性）
- [ ] 第 2 段起均包含「📎 上文衔接」行，概括前段核心结论
- [ ] 各段分析结构完整，无跨段信息混淆
- [ ] 末尾含「📊 全文综述」，涵盖主题脉络、共性翻译难点、语言特征总结
- [ ] 同一术语在不同段落的译法保持一致（或差异已在综述中说明）
