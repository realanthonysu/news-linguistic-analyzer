# 事实核查输出格式标准化

## 问题背景

SKILL.md 事实核查章节最初将「虚构标记词」放在第一层强制触发条件表中，与日期、人名+事件等并列。但虚构标记词的处理逻辑与其他条件不同——它不需要联网验证，而是直接标注内容属性。这导致 Agent 输出时可能混淆「⚠️ 事实核查」（已联网）和「⚠️ 内容属性提示」（无需联网）两种格式。

## SKILL.md 改造方案

**关键改动**：将「虚构标记词」从第一层强制触发条件表中移出，改为独立的「特殊路径」章节。

### 改造前（问题结构）

```markdown
### 第一层：强制触发（满足任一即触发 web search）
| 触发条件 | 说明 |
|---------|------|
| 虚构标记词 | 含 fiction, scenario, hypothetical, AI-generated 等标记 |
```

### 改造后（正确结构）

```markdown
### 第一层：强制触发（满足任一即触发 web search）
| 触发条件 | 说明 |
|---------|------|
| 含具体日期 | ... |
| 人名+重大事件组合 | ... |
| 伤亡/损失等敏感数据 | ... |
| 正在进行的重大事件 | ... |
| 与已知事实冲突 | ... |
（虚构标记词已移除）

### 特殊路径：虚构标记词（不触发 web search，直接标注）
当检测到 fiction、scenario、hypothetical、AI-generated 等虚构标记词时，
跳过 web search，直接标注：
「⚠️ 内容属性提示：...」

> 区分：「⚠️ 事实核查」= 已联网验证；「⚠️ 内容属性提示」= 检测到虚构标记词，无需联网。
```

## 格式 A：联网验证结果

**适用场景**：SKILL.md「第一层强制触发」条件满足（含具体日期、人名+重大事件、伤亡数据、重大事件进行中、与已知事实冲突），且 web search / web fetch 已执行。

**输出模板**：
```text
⚠️ 事实核查：经联网查询验证，[该事件为真实发生 / 该事件尚未发生 / 未找到相关权威报道]
   查询摘要：[1-2 句话概括搜索结果]
```

## 格式 B：内容属性提示

**适用场景**：检测到虚构标记词（`fiction`/`scenario`/`hypothetical`/`AI-generated`），跳过 web search 直接标注。

**输出模板**：
```text
⚠️ 内容属性提示：本段文本包含虚构标记词（如 hypothetical、AI-generated 等），可能为情景推演、创作练习或 AI 生成内容。请结合权威信源交叉验证后再作引用。
```

## 格式选择决策树

```
检测到虚构标记词（fiction/scenario/hypothetical/AI-generated）？
├── 是 → 格式 B（内容属性提示），跳过 web search
└── 否 → 是否满足强制触发条件（日期/人名+事件/伤亡数据等）？
          ├── 是 → 执行 web search → 格式 A（联网验证结果）
          └── 否 → 模型评估是否需要验证？
                    ├── 需要 → 执行 web search → 格式 A（联网验证结果）
                    └── 不需要 → 跳过事实核查，直接输出翻译
```
